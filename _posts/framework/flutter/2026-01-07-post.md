Flutter 실습을 하던 중 GetX 라우팅에 대해 배웠다.
<br/>

여기서 라우팅이라는 단어에 대해 의문이 생길수도 있는데 **라우팅(Routing)** 이란 네트워크에서 데이터 패킷이 출발지에서 목적지까지 도달하는 최적의 경로를 찾아주는 과정이라 한다.
<br/>
웹 개발에서는 URL 경로에 따라 응답을 보내는 방식(서버 사이드 라우팅)을 의미하기도 한다.
<br/>
<br/>
그외에도 아래와 같은 개념이 있다.
- 라우터 : 라우팅을 수행하는 네트워크 장비로, 여러 네트워크를 연결하고 패킷의 경로를 결정
- 패킷(Packet): 네트워크를 통해 전송되는 데이터의 단위
<br/>

### GetX 라우팅
우선 vsCode의 터미널에 flutter pub add get 이라는 명령어를 입력하여 Get을 다운받는다.
<br/>
그 후 pubspec.yaml 파일에 다운 받은 get이 버전과 함께 잘 명시되어있는 지 확인하면 된다

---
GetX는 Flutter에서 자주 사용되는 상태 관리 + 라우팅 + 의존성 관리를 한 번에 제공하는 라이브러리다.
<br/>
쉽게 설명하면 Flutter에서 화면은 카드 더미처럼 쌓인다. 새 화면으로 이동하면 2개의 화면이 쌓이는 건데 이걸 Navigation Stack이라고 한다. GetX의 모든 라우팅 메서드는 이 스택을 어떻게 조작하느냐의 차이다.

- Get.back()
    - 뒤로가기
    - 현재 화면을 pop하고 이전 화면으로 돌아감
    - Android 뒤로가기 버튼과 동일
    - 쌓여있던 카드를 하나 버리고 뒤 카드로 돌아가는 방식이라 생각하면 된다

- Get.to()
    - 화면 추가 (push)
    - 이전 화면 유지
    - 뒤로 가기 가능
    - 위젯 직접 전달 방식
    - 즉 새로운 카드 한장을 추가하는 방식이다.

- Get.toNamed()
    - 이름 기반 push
    - Get.to()와 스택 변화 동일
    - 라우트 이름을 사용
    - GetMaterialApp의 getPages 필요
    - 규모가 있는 앱에서의 화면 전환을 통해 관리하고 싶을 때 주로 사용한다.

- Get.off()
    - 현재 화면 제거 + 이동
    - 현재 화면만 제거하는 방식이다
    - 뒤로가기 불가 (다시 못 돌아감)
    - 로그인 성공 후 메인 화면으로, 화원가입 완료 후 홈화면으로 등에 사용
    - 카드하나를 버리고 새로운 카드 한장을 뽑는다고 생각하면 된다.

- Get.offNamed()
    - 이름 기반의 off
    - Get.off()와 스택 변화 동일
        - 현재 화면 제거 후 이동
    - 로그인 -> 홈 이동에서 자주 사용한다.

- Get.offAll()
    - 스택 전부 초기화
    - 모든 화면을 제거
    - 뒤로 가기 불가
    - 앱 상태 리셋
    - 로그아웃, 토큰 만료 등 상황에 사용
