그린고 그건 디자인 패턴

이 문서는 소프트웨어 디자인 패턴 중 하나를 설명합니다. 제목 그대로 "그린고 그건 디자인 패턴" 항목을 포함합니다.

디자인 패턴 선택: Singleton (싱글톤) 패턴

Singleton 패턴은 애플리케이션에서 단 하나의 인스턴스만 만들어야 할 때 사용합니다. 예: 설정(설정 값), 로깅, 데이터베이스 연결 풀 등의 전역적 상태 관리.

주요 목적

  • 클래스 인스턴스를 하나로 제한
  • 전역 접근점을 제공
  • 자원 낭비 예방

특징

  1. 생성자를 private 로 제한
  2. 정적 메서드를 통해 객체 반환
  3. 객체가 이미 생성되어 있으면 재사용

장점

  • 전역 상태를 중앙에서 관리
  • 인스턴스 생성 비용을 줄임
  • 동기화가 필요한 멀티스레드 환경에서 일관성 유지

단점

  • 단위 테스트가 어려워질 수 있음
  • 전역 상태가 의존성 숨김으로 이어질 수 있음
  • 과도하게 사용 시 유연성 저하

간단한 예 (자바 스타일 의사코드)

public class Singleton {
    private static Singleton instance;

    private Singleton() { }

    public static synchronized Singleton getInstance() {
      if (instance == null) {
        instance = new Singleton();
      }
      return instance;
    }
  }

이처럼 Singleton은 객체 생성 제어를 통해 애플리케이션 전역에 걸쳐 단일 인스턴스를 보장합니다.
