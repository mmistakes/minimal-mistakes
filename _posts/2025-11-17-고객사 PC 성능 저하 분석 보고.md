고객사 사용자 PC 성능 저하 심층 분석 (2025년 10월)

본 보고서는 2025년 10월 한 달간  사용자 PC에서 반복적으로 발생한 **시스템 성능 저하** 및 **응용프로그램 비정상 종료** 현상에 대한 근본 원인 분석과 구체적인 개선 방안을 다룹니다.

오렌지 에이전트 분석 결과, 총 **198개 유형의 증상**이 확인되었으며, 주요 원인은 운영체제 서비스의 자원 해제 불균형에서 비롯된 구조적인 문제로 판단되었습니다.

---

## 1. 증상 통계 및 주요 발생 유형

10월 한 달간 24대 PC를 분석한 결과, 가장 많은 빈도를 보인 증상 유형은 다음과 같습니다. 특정 유형에 편중되지 않고 자원 관리 불균형과 관련된 전반적인 문제가 확인되었습니다.

| 증상 유형 | 증상 유형의 개수 | 비율 (약) |
| :--- | :--- | :--- |
| **파일 시스템 처리 지연** | 34% |
| **프로그램 비정상 종료** | 22% |
| 다른 프로그램에 의한 강제 종료 | 21% |
| CPU 코어 1 개 이상 주기적 점유 | 21% |
| BSOD 발생 | 1.5% |

---

## 2. 주요 증상별 원인 및 조치 방안

가장 심각한 성능 저하를 유발한 4가지 핵심 원인과 그에 따른 해결책입니다.

### 2.1. Font Cache 문제: PresentationFontCache.exe → mscorwks.dll

| 구분 | 내용 |
| :--- | :--- |
| **증상** | CPU 코어 1개 이상 지속적 점유, 시스템 속도 저하 |
| **원인** | PresentationFontCache.exe(글꼴 캐시 서비스)가 .NET Framework의 `mscorwks.dll` 모듈을 사용할 때, **캐시 파일 손상** 또는 **메모리 해제 루틴 실패**로 인해 폰트 캐시가 반복 재생성되면서 CPU 점유율이 급증했습니다. 노후된 그래픽 드라이버(2021년 버전) 사용이 문제와 연관된 것으로 파악됩니다. |
| **조치 방안** | 1) **Intel 최신 그래픽 드라이버 (DCH 버전) 설치**를 통해 폰트 렌더링 과정에서 발생하는 오류를 근본적으로 해결하고 재발을 방지합니다. |

### 2.2. Windows 서비스 과부하: svchost.exe → sechost.dll

| 구분 | 내용 |
| :--- | :--- |
| **증상** | CPU 코어 1개 이상 지속적 점유, 시스템 응답 지연 |
| **원인** | `svchost.exe` 내에서 실행되는 특정 서비스(예: SysMain)가 **Windows Update 스캔 반복** 또는 **서비스 캐시 손상**으로 인해 비정상적으로 반복 호출되었습니다. 이 서비스 제어 요청이 `sechost.dll`에 과도하게 집중되어 CPU 점유율이 멈추지 않고 상승했습니다. |
| **조치 방안** | 1) **Windows 서비스 재등록 및 초기화** (Windows Update 구성 요소 초기화)를 진행하여 임시 캐시 파일을 삭제했습니다. (명령어: `net stop wuauserv`, `net stop bits`, `rd /s /q %windir%\SoftwareDistribution`, `net start wuauserv`, `net start bits`) |
| **조치 결과** | 11월 7일 조치 후, **동일 증상은 재발하지 않았습니다.** |

### 2.3. 그래픽 커널 문제: System → dxgmms2.sys

| 구분 | 내용 |
| :--- | :--- |
| **증상** | CPU 코어 1개 이상 주기적 점유, 화면 깜빡임/멈춤, 그래픽 지연 |
| **원인** | 시스템 커널 프로세스에 로드된 `dxgmms2.sys`(DirectX 그래픽 메모리 관리 시스템)가 **그래픽 메모리를 효율적으로 해제하지 못해** CPU 자원을 반복 점유했습니다. Windows OS 버전(2023년 출시)과 그래픽 드라이버 버전(2021년 출시) 간의 **버전 불일치**로 인한 호환성 문제가 원인으로 파악됩니다. |
| **조치 방안** | 1) **Intel 최신 그래픽 드라이버 설치**를 통해 OS와 그래픽 자원 관리 모듈 간의 호환성 문제를 해결하고 시스템 응답성을 개선합니다. |

### 2.4. 보안 모듈 충돌: Hwp.exe → A3Dll32.dll (한글)

| 구분 | 내용 |
| :--- | :--- |
| **증상** | 한글 프로그램(`Hwp.exe`) 비정상 종료 |
| **원인** | 한글 프로그램에 로드된 Blackfort Security 사의 보안 모듈(`A3Dll32.dll`)이 외부 공격 차단을 위해 프로그램 내부 코드를 감시(후킹)하는 과정에서 **메모리 접근 오류**가 발생하여 한글 프로그램이 강제로 종료되었습니다. |
| **조치 방안** | 1) **한글 프로그램 설정에서 실시간 악성 코드 검사 항목을 비활성화**하여 보안 모듈과의 충돌 가능성을 제거합니다. |

---

## 3. 최종 결론 및 재발 방지 권고

분석 결과, 고객사 사용자 PC에서 발생한 시스템 성능 문제는 **노후된 시스템 구성 요소(특히 그래픽 드라이버)** 사용으로 인한 구조적 불안정성에서 비롯된 것으로 결론지었습니다.

| 주요 원인 | 권장 조치 |
| :--- | :--- |
| 운영체제 서비스 과부하 | Windows 서비스 초기화 및 정기적인 재부팅 |
| 그래픽 드라이버 오류 | Intel 최신 그래픽 드라이버 (DCH 버전) 업데이트 |
| 보안 모듈 충돌 | 한글 프로그램 내 실시간 악성 코드 검사 기능 비활성화 |
| .NET 글꼴 캐시 손상 | 최신 그래픽 드라이버 업데이트를 통한 렌더링 오류 방지 |

**지속적인 시스템 구성 요소 업데이트와 정기적인 서비스 관리**를 통해 동일 증상의 재발을 방지하고 전반적인 시스템 안정성을 강화해야 합니다.
