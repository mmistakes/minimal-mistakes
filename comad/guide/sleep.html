Sleep — 메모리 정리 에이전트

Claude Code의 프로젝트별 auto memory 디렉토리를 전수 스캔해 중복 감지, 통합, 오래된 기록 정리를 수행합니다. "dream" 한 마디로 발동되며, 변경 이력을 MCP 서버로 조회할 수 있습니다.

Bun MCP server auto memory cross-project

개요

Claude Code는 각 프로젝트마다 ~/.claude/projects/<cwd>/memory/에 자동 메모리를 저장합니다. 시간이 쌓이면 중복, 오래된 정보, 파편화된 내용이 축적되는데, Sleep이 이를 주기적으로 정리합니다.

Before sleep:
memory/project_session_20260412.md   (62 lines)
memory/project_session_20260408.md   (70 lines)
memory/feedback_ci_check.md          (동일 내용)
memory/feedback_ci_after_push.md     (거의 동일)
MEMORY.md                            (1,333 lines, 많은 링크 깨짐)

After sleep (dream):
memory/project_session_20260412.md   (5 lines, 요약본)
memory/project_session_20260408.md   (6 lines)
feedback_ci_check.md 삭제
feedback_ci_after_push.md            (통합 버전)
MEMORY.md                            (1,166 lines, 링크 전수 검증)

메모리 유형

자동 메모리는 4가지로 분류됩니다. Sleep은 유형별로 다른 정리 정책을 적용합니다.

유형내용정리 정책
user사용자의 역할, 선호, 지식 수준거의 변하지 않음. 중복만 제거
feedback사용자가 준 피드백 (하지 마/계속 이렇게 해)유사 항목 통합, 왜 & 언제 적용 명시
project진행 중인 작업, 목표, 데드라인, 기술 결정시간 지난 세션 요약 압축, 최근 3-5개만 상세 유지
reference외부 시스템 포인터 (Linear 프로젝트 URL 등)링크 깨짐 확인, 중복 제거

dream 실행

아무 Claude Code 세션에서 "dream" 또는 "정리해줘" 등 트리거 문구를 입력하면 comad-sleep 에이전트가 발동됩니다.

MEMORY.md 인덱스

각 프로젝트의 memory/MEMORY.md는 세부 메모리 파일들을 한 줄 요약으로 엮은 인덱스입니다. Sleep이 항상 이 인덱스의 일관성을 유지합니다.

# memory/MEMORY.md
- [Session 2026-04-13](project_session_20260413.md) — GraphRAG recall 93%, 크론 OAuth 해결, LaunchAgent 마이그레이션
- [Session 2026-04-12](project_session_20260412.md) — 자가진화 6 Phase, 스코어 84/100
- [Pending tasks](project_pending_tasks.md) — meta-edge 분할 · 50K 그래프 성장

- ...

MCP 서버 — 조회

comad-sleep은 MCP 서버도 제공합니다. 정리 이력과 메타데이터를 대화로 조회.

도구용도
comad_sleep_info마지막 dream 실행 시각, 다음 권장 실행 일정
comad_sleep_history최근 N회의 dream 리포트 (무엇이 바뀌었나)

정리 정책

Sleep이 자동으로 판단하는 규칙들:

무엇을 저장하지 말 것 (중복되는 기억)

  • 코드 패턴/컨벤션/아키텍처 — git log/소스가 진실
  • 디버깅 해결책 — 커밋 메시지에 이미 있음
  • 현재 대화 중인 task 상태 — 일시적 정보
  • CLAUDE.md에 이미 명시된 것

세션 파일 압축 타이밍

  • 같은 작업의 후속 세션이 있으면 이전 세션을 5-10줄로 압축 (링크와 핵심 결정만)
  • 30일 이상 지난 세션은 자동 요약 후보
  • "How to apply" 섹션이 있는 기억은 TTL이 명시된 경우에만 삭제
트리거 자동화 — SessionStart hook이 MEMORY.md 라인 수를 확인해 150줄 초과면 "dream"을 제안합니다. 직접 말 안 해도 Claude가 먼저 안내.
