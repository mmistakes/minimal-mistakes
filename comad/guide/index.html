업그레이드 Upgrading

이미 설치했다면 comad 한 명령으로 main + 6개 모듈 + deps + agents를 한 번에 업데이트합니다. Dirty tree abort · 자동 스냅샷 · 롤백 지원.

# Any directory (symlinked to ~/.local/bin/comad by install.sh)
comad status              # VERSION + module SHAs (dirty flagged)
comad upgrade --dry-run   # preview changes
comad upgrade             # pull + deps + agents + auto-backup
comad backups             # list snapshots
comad rollback <ts>       # restore a prior snapshot

# Or via Makefile from the repo root
make test                 # full suite (brain + eye)
make clean                # dry-run: preview 300MB+ of caches & logs
make clean-apply          # actually delete them
make clean-deep           # also nuke node_modules / .venv (~3.6 GB)
make validate-config      # comad.config.yaml + presets vs JSON Schema

루트 VERSION (semver) + comad.lock (6개 모듈 branch+SHA 고정)으로 package-lock.json처럼 조합을 재현 가능. 기존 설치 사용자는 한 번 git pull origin main && ./install.sh 하면 comad 명령이 생깁니다. 세부: README § Upgrading.

📄 Config 계약 — ADR 0002: comad.config.yaml의 JSON Schema(schema/comad.config.schema.json)를 기준으로 필수·선택 섹션, 타입, URL 패턴, Neo4j URI 등 자동 검증. 오타나 잘못된 타입은 CI의 config-schema-validation 잡이 2초 안에 잡아냅니다. make validate-config로 로컬에서도 실행.

📐 저장소 전략 — ADR 0001: umbrella는 배선(installer / scripts / Makefile / VERSION / comad.lock)을 소유, 각 모듈 디렉터리는 자기 .git이 오너. Structure Guard CI가 PR마다 모듈 경계를 자동 검증 — 중복 파일·빌드 산출물 트래킹·VERSION 형식 오류를 막아줍니다.

📂 경로 무관 — 어느 폴더, 어느 이름으로 클론해도 동작합니다. scripts/comad가 자기 심링크를 따라 repo root를 찾고, scripts/render-templates.sh가 {{COMAD_ROOT}} 플레이스홀더를 install/upgrade 시점에 절대경로로 치환합니다. ~/Desktop/내-에이전트, /opt/comad 어디든 OK.
