## Episode Notes

### Codex 01. Codex를 코드 생성기가 아니라 작업 수행 에이전트로 보기

- 역할: 시리즈의 관점 설정
- 핵심 메시지: Codex는 답변 생성 도구가 아니라 저장소를 읽고, 수정하고, 검증하는 작업 수행 주체로 다룬다.
- 포함 항목: 작업 수행 에이전트 관점, 프롬프트와 실행 작업의 차이, 사람이 책임져야 하는 목표·경계·승인·검토
- 산출물 후보: Codex 작업 요청 기본 틀, 사람이 결정해야 할 항목 체크리스트

### Codex 02. 왜 Codex에는 하네스가 필요한가

- 역할: 하네스 개념을 Codex 운영 문제로 연결
- 핵심 메시지: Codex의 결과를 안정화하려면 좋은 요청만으로는 부족하고, 프로젝트 규칙과 검증 구조가 필요하다.
- 포함 항목: Codex 작업 편차의 원인, 지시 파일과 실행 환경의 차이, 검증 가능한 규칙과 자연어 원칙의 차이
- 산출물 후보: Codex 하네스 구성 지도, 규칙 분류표

### Codex 03. Codex 첫 작업 요청 작성법

- 역할: 실전 진입점
- 핵심 메시지: 첫 요청에는 목표, 범위, 금지 사항, 완료 기준, 검증 방법이 들어가야 한다.
- 포함 항목: `Goal`, `Scope`, `Constraints`, `Definition of Done`, `Verification`, 보고 형식
- 산출물 후보: 첫 작업 요청 템플릿, 작업 전 누락 정보 점검표

### Codex 04. Codex용 `AGENTS.md` 작성법

- 역할: 프로젝트 지시 파일의 기본 설계
- 핵심 메시지: `AGENTS.md`는 작업 요청의 대체물이 아니라 프로젝트 운영 기준의 진입점이다.
- 포함 항목: 저장소 목적, 우선 확인 경로, 작업 원칙, 검증 기준, 파일 수정 경계, 리뷰 체크포인트
- 산출물 후보: `AGENTS.md` 기본 골격, 저장소 유형별 항목표

### Codex 05. 좋은 `AGENTS.md`는 왜 짧아야 하는가

- 역할: `AGENTS.md` 비대화 방지
- 핵심 메시지: 긴 `AGENTS.md`는 토큰 비용, 중복 지시, 책임 혼합을 키우므로 밖으로 뺄 내용을 분리해야 한다.
- 포함 항목: 지시 파일 비대화, 토큰 낭비, 상세 규칙의 외부 문서 분리, 자동화 가능한 규칙의 config 이동, 반복 절차의 skill 이동
- 산출물 후보: `AGENTS.md` 축약 체크리스트, 문서 분리 판단표

### Codex 06. Codex를 plan-first로 운영하기

- 역할: 작업 실행 흐름 설계
- 핵심 메시지: 복잡한 작업은 바로 수정하지 않고 범위, 위험, 검증 방법을 먼저 계획하게 해야 한다.
- 포함 항목: plan-first가 필요한 작업, 계획 필드, 작업 중 계획 갱신, 승인 지점, 계획과 결과 보고의 차이
- 산출물 후보: plan-first 요청 템플릿, 작업 계획 상태 필드 예시

### Codex 07. config로 Codex 작업 일관성 확보하기

- 역할: 운영 기준의 설정 계층 분리
- 핵심 메시지: 반복되는 실행 환경, 권한, 검증 명령은 자연어 지시보다 config로 고정하는 편이 안정적이다.
- 포함 항목: 자연어 지시와 config의 책임 차이, 실행 환경 고정, 검증 명령 고정, 권한과 승인 경계, 프로젝트별 기본값
- 산출물 후보: Codex 운영 config 점검표, 반복 지시를 config 후보로 분류하는 표

### Codex 08. skill로 Codex 반복 작업 표준화하기

- 역할: 반복 절차의 재사용 계층 설계
- 핵심 메시지: 반복되는 작성, 검토, 릴리스, 점검 절차는 skill로 분리한다.
- 포함 항목: skill이 적합한 작업, `AGENTS.md`와 skill의 책임 분리, skill 문서의 최소 구성, 입력과 출력 형식, 검증 절차
- 산출물 후보: skill 설계 템플릿, `AGENTS.md`에서 skill로 옮길 규칙 후보 목록

### Codex 09. Codex subagent는 언제 써야 하는가

- 역할: 병렬 작업과 역할 분리 기준
- 핵심 메시지: subagent는 병렬성과 전문화가 실제 이득을 줄 때만 사용하고, 기본값은 단일 흐름으로 둔다.
- 포함 항목: subagent 사용 조건, 단일 에이전트 기준선, 병렬 탐색과 병렬 수정의 차이, 파일 소유권 분리, 결과 통합과 검증 책임
- 산출물 후보: subagent 사용 판단표, subagent 작업 위임 템플릿, 결과 통합 체크리스트

### Codex 10. Codex 프로젝트 적용용 운영 템플릿

- 역할: 시리즈의 종합 적용편
- 핵심 메시지: 작업 요청, `AGENTS.md`, plan-first, config, skill, subagent 기준을 하나의 프로젝트 운영 템플릿으로 묶는다.
- 포함 항목: 도입 전 점검, 첫 작업 요청, `AGENTS.md` 최소안, plan-first 운영 규칙, config 후보, skill 후보, subagent 사용 기준, 검증과 리뷰 루프
- 산출물 후보: Codex 프로젝트 운영 템플릿, `AGENTS.md` 최소 템플릿, 첫 작업 요청 템플릿, plan-first 체크리스트

## Planning Notes

- 1~2편은 관점과 이론을 포함하되, 기존 하네스 엔지니어링 입문 내용을 반복하지 않는다.
- 3편부터는 실전 활용과 프로젝트 운영 중심으로 구성한다.
- 각 글은 `Codex` 중심으로 쓰고, 다른 도구는 필요한 경우 경계 설명에만 제한적으로 언급한다.
- 특정 도구를 과장하거나 찬양하는 표현은 피한다.
- 실제 발행을 시작하면 각 글마다 `검증 기준일`, `테스트 환경`, `테스트 버전`, `직접 재현한 결과`, `한계와 예외`를 채운다.
- 최신성에 민감한 Codex 기능, CLI 동작, 설정, 문서 링크는 발행 시점에 공식 문서 기준으로 재검증한다.
