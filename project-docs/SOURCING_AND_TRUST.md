# Sourcing And Trust

## 목적

- 이 문서는 블로그 글의 신뢰도를 높이기 위한 출처 표기 기준을 정리한다.
- 글 끝 참고문헌만으로 끝내지 않고, 본문의 주장 옆에 근거를 붙이는 것을 기본 원칙으로 한다.

## 기본 원칙

- 주장마다 근거를 연결한다.
- 최신성이 중요한 내용은 시점을 함께 적는다.
- 사실과 의견을 분리한다.
- 재현 가능한 정보는 가능한 한 본문 초반에 모아둔다.
- 개인 블로그를 또 다른 개인 블로그로 뒷받침하지 않는다.

## 출처 우선순위

신뢰도는 아래 순서를 기본으로 본다.

1. 공식 문서
2. 논문
3. 표준 문서
4. 원저자 발표 또는 저장소
5. 공신력 있는 기관 자료
6. 기사나 2차 정리글

- 가능하면 같은 주장에는 가장 높은 등급의 출처를 우선 사용한다.
- 공식 문서가 없는 경우에만 2차 자료를 보조적으로 쓴다.
- 2차 자료를 쓸 때는 왜 그 자료를 썼는지 문맥상 드러나게 적는다.

## 본문 표기 방식

- 사실 주장에는 바로 다음 문장이나 같은 문단 안에서 `근거:` 또는 `공식 문서 기준:` 형태로 링크를 붙인다.
- 해석, 실무 감각, 개인적 판단에는 `의견:` 또는 `내 해석:`을 붙여 사실과 분리한다.
- 샘플 분석에서 직접 본 값은 `직접 확인:` 또는 `샘플 관찰:`로 표시한다.

예시:

```md
OpenAI Codex 공식 문서는 `AGENTS.md`를 작업 전에 읽는 지침 파일로 설명한다.
근거: [Custom instructions with AGENTS.md](https://developers.openai.com/codex/guides/agents-md)

의견: 이 글에서는 이런 여러 메커니즘을 묶어 `harness engineering`이라는 설명 틀로 다룬다.
```

## 시점 표기

- AI 도구, 서비스 기능, 정책, 가격, 명령어, UI, 운영 흐름처럼 바뀌기 쉬운 내용은 `YYYY-MM-DD 기준` 또는 `YYYY년 M월 기준`을 적는다.
- 시점은 본문 초반의 `검증 기준`, `재현 환경`, `주의` 섹션에 넣는 것을 기본으로 한다.

예시:

```md
- 시점: 2026-04-15 기준 VS Code, OpenAI Codex, Claude Code 공식 문서 확인
```

## 재현 정보

설정/실습 글은 아래 항목을 우선 공개한다.

- 운영체제
- 핵심 도구 이름
- 버전 또는 버전을 고정하지 않았다는 사실
- 실행 명령어
- 확인한 화면이나 결과

분석 글은 아래 항목을 우선 공개한다.

- 샘플 식별자
- 해시
- 파일 크기
- 사용 도구
- 분석 절차
- 동작 조건

## 글 끝 참고문헌

- 글 끝의 `출처 및 참고` 섹션은 유지한다.
- 다만 이 섹션은 본문 근거를 모아두는 인덱스 역할이어야 하며, 본문 근거를 대체하지 않는다.
- 본문에서 이미 연결한 링크를 글 끝에서 다시 정리해도 된다.
