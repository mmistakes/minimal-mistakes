The Spark

시작: 하나의 불편함

GeekNews에서 흥미로운 기사를 발견한다. 브라우저 탭에 띄워둔다. 나중에 정리하려고 한다. 정리하지 않는다. 한 달 뒤 같은 기사를 또 발견한다. "이거 어디서 본 것 같은데..." — 프로젝트를 시작하게 된 실제 경험

수집 → 구조화 → 분석 → 실행. 이 파이프라인이 끊기는 지점마다 에이전트를 하나씩 만들었습니다. 기사 하나를 발견하면 자동으로 아카이브되고, 지식 그래프에 연결되고, 시뮬레이션 분석까지 흐르는 시스템. 그것이 COMAD입니다.

COMAD 없이

기사를 읽고 잊는다. 같은 주제를 반복 검색한다. 연결 고리를 머릿속에만 둔다. 분석은 "느낌"에 의존. 사진 보정은 매번 처음부터.

COMAD와 함께

기사가 자동 수집, 61K 노드 그래프에 연결, 10개 렌즈로 분석, 사진은 33줄 에이전트가 보정. 모든 맥락이 축적되고, 잠자는 동안 메모리가 정리됨.

The Problem

AI 도구는 넘쳐나지만, 통합된 생태계는 없었습니다

01

파편화된 도구들

뉴스 수집, 지식 관리, 분석, 사진 보정 — 각각 별도 도구를 쓰면 컨텍스트가 끊깁니다.

02

수동 연결의 한계

도구 간 데이터를 사람이 옮기면 자동화의 의미가 사라집니다.

03

외부 의존성 리스크

프레임워크를 통째로 쓰면 유지보수 부담. 아이디어만 차용, 자체 구현.

Methodology

사람은 설계, AI는 실행

사람 (설계자)

  • 시스템 아키텍처 설계
  • 어떤 문제를 풀 것인지 결정
  • 품질 기준 설정 (MAE 20, 필독 13%)
  • 차용할 아이디어 선정
  • 실전 테스트 결과 판단

AI (Claude Code)

  • 코드 구현 + 테스트 작성
  • 구체적인 해법 코딩
  • 기준에 맞는 최적화 실행
  • 차용 포인트 분석 + 자체 구현
  • 테스트 자동화 + 배치 실행
